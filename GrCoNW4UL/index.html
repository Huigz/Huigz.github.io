Database概论笔记 - SQL_1

SQL

SQL : Structured Query Language

关于SQL

Three relational languages

  1. QUEL
  2. QBE
  3. SQL

SQL是目前Relational DB的标准语言

SQL是一种 Very-High-Level language

SQL的结构

SQL的组成

  • Table（relation）
  • Row（tuple)
  • Column (attribute)

其中 Table 包含了

  • Base Table
  • View

基本命令

  • Data Definition Language（DDL）

    Define Data

  • Data Manipulation language （DML）

    操纵 Data

  • Data Contral Language （DCL）

DDL 用于定义schemas，tables，views

  • CREATAE
  • DROP
  • ALTER

DML用于操纵数据 搜索，更新 Table中的数据

  • SELECT（FROM WHERE）
  • INSERT
  • DELETE
  • UPDATE

DDL-创建Table

Using CREATE TABLE

CREATE TABLE R
(	A1:D1
	A2:D2
	... ...
	An:Dn
 	Constraints
)

  • R: Relation Name

  • A: Attrbute Name

  • D: Domain (= data type) of A

  • Constraints：制约条件

通过 CREATE TABLE创建的Relation 称之为base Table

  • relation（and its tuple）都是物理储存
  • 为空表
  • Attritubes 根据 CREATE TABLE指令的顺序创建

Attritube Data Type

System Defined

  • 数值型：INT,SMALLINT,FLOAT(or REAL)...
  • 字符串：CHAR(n),VARCHAR(n),CLOB(大容量文章)...
  • bit string(数据流)：BIT(n),BIT VARYING(n),BLOB(大容量数据)
  • Bool: TRUE,FLASE,UNKOWN
  • Date: YEAR,MONTH,DAY

User Defined

用户自定义数据类型

CREATE DOMAIN SSN-TYPE AS CHAR(9)

Constraints 制约条件

  • PRIMARY KEY
  • UNIQUE
  • NOT NULL
  • FOREIGN KEY REFERENCES
  • ...

Key/Entity Constraints

ER:

Relation: EMP(SSN,Name,Bdate)

Table (SQL):

CREATE TABLE EMP(
	SSN CHAR(9) NOT NULL
	Name VARCHAR(10) NOT NULL
    Bdate DATE
    PRIMARY KEY(SSN)
    UNIQUE(Name)
)

​ PK-> SSN KEY->Name

Referential Constraints

ER:

Relation:

​ EMP(SSN,Name,Bdate)

​ DEPT(DNO,Name,Mgr_SSN)

Table:

CREATE TABLE EMP(
	SSN CHAR(9) NOT NULL
	Name VARCHAR(10) NOT NULL
    Bdate DATE
    PRIMARY KEY(SSN)
    UNIQUE(Name)
)

CREATE TABLE DEPT(
	DNO INT NOT NULL
	Name CHAR(10)
    Mgr_SSN CHAR(9) NOT NULL
    PRIMARY KEY(DNO)
    FOREIGN KEY(Mgr_SSN) REFERENCES EMP(SSN)
)

Default/Domain/Check

Default（默认值）

CREATE TABLE DEPT(
	Mgr_SSN CHAR(9) DEFAULT 999999999
)

Check

给value设定特殊条件

DNO INT CHECK(DNO>0) AND (DNO<20)

Domain (Data Type)

CREATE DOMAIN SSN_TYPE AS CHAR(9)
CREATE TABLE EMP(
	SSN SSN_TYPE NOT NULL
	name CHAR(10) NOT NULL
 	bdate DATE
  	PRIMARY KEY (SSN)
    UNIQUE (name)
)

CREATE TABLE DEPT(
	DNO INT NOT NULL
	Name CHAR(10)
    Mgr_SSN SSN_TYPE NOT NULL
    PRIMARY KEY(DNO)
    FOREIGN KEY(Mgr_SSN) REFERENCES EMP(SSN)
)

如果SSN的Data Type需要改变 可直接改变SSN_TYPE的定义

Trigger 触发

CREATE DOMAIN SSN_TYPE AS CHAR(9)
CREATE TABLE EMP(
	SSN SSN_TYPE NOT NULL
	name CHAR(10) NOT NULL
 	bdate DATE
  	PRIMARY KEY (SSN)
    UNIQUE (name)
    ON DELETE  SET NULL
    ON UPDATE  CASCADE
)

CREATE TABLE DEPT(
	DNO INT NOT NULL
	Name CHAR(10)
    Mgr_SSN SSN_TYPE NOT NULL
    PRIMARY KEY(DNO)
    FOREIGN KEY(Mgr_SSN) REFERENCES EMP(SSN)
)

条件：

  • ON ... (触发条件)

操作：

  • SET ... (设定值)
  • CASCADE (对Referential Data采取相同的操作)

Ex.

ER:

Relation:

​ EMP(SSN,NAME,BDATE,DNO,SUPER_SSN)

​ DEP(DNO,NAME,MGR_SSN)

TABLE:

DDL-删除Table

Using DROP TABLE

Have 2 options:

  • CASCADE
  • RESTRICT

RESTRICT

只有当要被删除的表 没有任何Referenced的constraints或View时 才会操作删除任务

CASCADE

所有Referenced的表或View都会被一同删除

DDL-变更Table (Column)

Using ALERT TABLE

用于 添加 删除 变更 Table的 Column

Eg.

EMP表中 添加新的Column hobby

ALERT TABLE EMP ADD COLUMN hobby CHAR(12)

EMP表中 删除Column bdate

ALERT TABLE EMP DROP COLUMN bdate CASCADE
  • 除了CASCADE 也可以使用 RESTRICT 来确保独立删除

关于DDL

综上所述，DDL的操作是相对于 表结构 的操作
