Melon music PC웹사이트 Crawling통해 Artists의 곡 참여 연결망 생성과 분석

사전 분석

Melon music의 웹사이트 HTTP Stream분석 통해 API확인

멜론의 아티스트 페이지 주소는 https://www.melon.com/artist

아티스트의 곡 확인하려고 해서 https://www.melon.com/artist/song.htm 사용 [중간 빨간 화살표]

Params에 Artist의 ID 부여해야 한다. [파란 화살표]

곡 정보는 Table구조통해 밑 [빨간화살표]에 전시

연결망 만들기 위해 Artist의 발매 곡 정보아니라 참여 정보 필요하다.

여기에 멜론은 JS기술 사용해서 데이터 동작적으로 Get하고 표시되는 것이다.

HTTP Stream분석하면:

songpaging.htm에서 listType라는 Param통해 곡 정보 장르 조정하는 것 볼 수 있다.

발매의 값은 F다.

songpaging.htm의 Response확인:

곡명 보면 선정된 Artist(아이유)가 참여했는 곡 확인할 수 있고 곡 원작의 Artist도 확인할 수 있다.

특이 [빨간 라인]보면 Artist의 ID도 Source에 표시되고 있다.

Crawling의 절차

  1. 초기 Artist선택
  2. Artist ID Crawling Queue에 넣다
  3. Queue에 있는 Artist Crawling -> 곡명, 원작 데이터 받기
  4. New Artist ID(참여 곡의 원작) Crawling Queue에 넣다

Loop(step3, step4 | Until Queue is empty)

언어: Python3

플랫품: Scrapy

Source

https://github.com/Huigz/melonSpider

Result

running:

scrapy crawl melonSpider -o melon.json
